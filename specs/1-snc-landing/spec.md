# Feature Specification: S&C 신차 장기 렌트 리스 공개 웹사이트

**Feature Branch**: `1-snc-landing`
**Created**: 2026-02-04
**Status**: Draft
**Input**: S&C 신차 장기 렌트 리스 - 운영자가 판매 차량 정보와 출고 내역을 직접 관리할 수 있는 관리자 페이지(CMS) + 공개 웹사이트 구축. 고객은 웹사이트에서 최신 판매 차량과 출고 사례를 확인하고 문의할 수 있음.

## Clarifications

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 판매 차량 탐색 (Priority: P1)

고객이 S&C 웹사이트에 방문하여 국산/수입 필터와 제조사 필터를 활용해 원하는 판매 차량을 탐색하고, 차량별 렌트/리스 가격 정보를 확인한다.

**Why this priority**: 핵심 비즈니스 기능으로, 고객이 차량 정보를 확인하지 못하면 상담 전환이 불가능하다.

**Independent Test**: 웹사이트 접속 후 필터 조작과 차량 카드 확인만으로 완전히 테스트 가능. 고객에게 차량 정보 제공이라는 독립적 가치를 전달한다.

**Acceptance Scenarios**:

1. **Given** 고객이 랜딩 페이지에 접속했을 때, **When** 판매 차량 섹션으로 이동하면, **Then** 전체 판매 차량 목록이 카드 형태로 표시된다.
2. **Given** 판매 차량 목록이 표시된 상태에서, **When** "국산차" 필터를 선택하면, **Then** 국산 제조사의 차량만 표시된다.
3. **Given** 판매 차량 목록이 표시된 상태에서, **When** "수입차" 필터를 선택하면, **Then** 수입 제조사의 차량만 표시된다.
4. **Given** 1차 필터가 적용된 상태에서, **When** 특정 제조사 로고를 선택하면, **Then** 해당 제조사의 차량만 표시된다.
5. **Given** 차량 카드가 표시될 때, **When** 가격 정보가 있으면, **Then** "렌트 1,234,000원" 형식으로 표시된다.
6. **Given** 차량 카드가 표시될 때, **When** 가격 정보가 없으면, **Then** "렌트 비용문의" 형식으로 표시된다.
7. **Given** 차량에 뱃지가 있을 때, **When** 카드가 렌더링되면, **Then** "즉시출고", "프로모션" 등의 뱃지가 스티커 형태로 표시된다.

### User Story 2 - 견적 상담 신청 (Priority: P1)

고객이 관심 있는 차량에 대해 견적 상담을 신청한다. 필수 정보(이름, 연락처, 지역, 유형, 초기자금, 계약기간)를 입력하고 개인정보 동의 후 제출한다.

**Why this priority**: 비즈니스 전환(리드 수집)의 핵심 기능. 차량 탐색과 함께 MVP의 필수 요소이다.

**Independent Test**: 모달 열기, 폼 입력, 제출까지 독립적으로 테스트 가능. 리드 수집이라는 핵심 비즈니스 가치를 전달한다.

**Acceptance Scenarios**:

1. **Given** 헤더의 "빠른 상담" 버튼이 표시된 상태에서, **When** 버튼을 클릭하면, **Then** 견적 상담 모달이 열린다.
2. **Given** 차량 카드의 "간편 상담 신청" 버튼이 표시된 상태에서, **When** 버튼을 클릭하면, **Then** 해당 차량 정보가 자동으로 세팅된 모달이 열린다.
3. **Given** 모달이 열린 상태에서, **When** 필수 입력 필드를 모두 채우고 개인정보 동의를 체크한 후 제출하면, **Then** 접수 완료 메시지가 표시된다.
4. **Given** 모달이 열린 상태에서, **When** 필수 필드 중 하나라도 비어있으면, **Then** 해당 필드에 대한 검증 오류가 표시된다.
5. **Given** 전화번호 입력 시, **When** 숫자와 하이픈을 섞어 입력하면, **Then** 시스템이 숫자만 추출하여 저장한다.
6. **Given** 제출 버튼을 클릭한 상태에서, **When** 처리 중일 때, **Then** 버튼이 비활성화되어 중복 제출을 방지한다.

### User Story 3 - 랜딩 페이지 첫 인상 (Priority: P2)

고객이 웹사이트에 처음 접속했을 때 히어로 배너를 통해 S&C의 서비스를 한눈에 파악하고 신뢰감을 느낀다.

**Why this priority**: 첫 인상과 브랜드 신뢰 구축에 중요하지만, 차량 탐색과 상담 신청보다는 우선순위가 낮다.

**Independent Test**: 페이지 접속만으로 히어로 섹션 확인 가능. 브랜드 인지도 전달이라는 독립적 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 고객이 웹사이트에 접속하면, **When** 페이지가 로드되면, **Then** 히어로 배너가 최상단에 표시된다.
2. **Given** 히어로 배너가 표시된 상태에서, **When** CTA 버튼을 클릭하면, **Then** 견적 상담 모달이 열린다.

---

### User Story 4 - 출고 내역 확인 (Priority: P2)

고객이 최근 출고된 차량 내역을 확인하여 S&C의 실적과 신뢰도를 확인한다.

**Why this priority**: 사회적 증거(social proof)로 신뢰 구축에 도움이 되지만, 핵심 전환 기능은 아니다.

**Independent Test**: 출고 내역 섹션 확인만으로 테스트 가능. 신뢰도 향상이라는 독립적 가치를 전달한다.

**Acceptance Scenarios**:

1. **Given** 출고 내역 섹션에 도달하면, **When** 데이터가 있으면, **Then** 최대 6개의 출고 차량이 2x3 그리드로 표시된다.
2. **Given** 출고 차량 카드가 표시될 때, **When** 카드를 확인하면, **Then** 차량 이미지, 차량명, 출고일이 표시된다.

---

### User Story 5 - 핵심 강점 확인 (Priority: P3)

고객이 S&C의 핵심 강점과 차별점을 확인하여 서비스 선택에 대한 확신을 얻는다.

**Why this priority**: 마케팅 메시지 전달 역할로, 필수 기능은 아니지만 전환율 향상에 기여한다.

**Independent Test**: 강점 섹션 스크롤만으로 테스트 가능. 차별화 포인트 전달이라는 독립적 가치를 제공한다.

**Acceptance Scenarios**:

1. **Given** 핵심 강점 섹션에 도달하면, **When** 섹션이 렌더링되면, **Then** 강점 카드들이 이미지, 제목, 설명과 함께 표시된다.

---

### User Story 6 - FAQ 확인 (Priority: P3)

고객이 자주 묻는 질문을 확인하여 궁금증을 해소한다.

**Why this priority**: 고객 셀프서비스로 문의 감소에 기여하지만, 핵심 전환 기능은 아니다.

**Independent Test**: FAQ 섹션 확인 및 아코디언 동작만으로 테스트 가능.

**Acceptance Scenarios**:

1. **Given** FAQ 섹션에 도달하면, **When** 질문 항목을 클릭하면, **Then** 해당 답변이 펼쳐진다.
2. **Given** 하나의 FAQ가 펼쳐진 상태에서, **When** 다른 질문을 클릭하면, **Then** 기존 답변은 닫히고 새로운 답변이 펼쳐진다.

---

### User Story 7 - 네비게이션 (Priority: P2)

고객이 헤더의 네비게이션을 통해 원하는 섹션으로 빠르게 이동하거나 전화 상담을 요청한다.

**Why this priority**: 사용성 향상에 중요하지만, 스크롤로 대체 가능하여 핵심 기능보다 우선순위가 낮다.

**Independent Test**: 헤더 버튼 클릭만으로 테스트 가능. 편의성이라는 독립적 가치를 전달한다.

**Acceptance Scenarios**:

1. **Given** 헤더가 표시된 상태에서, **When** 로고를 클릭하면, **Then** 페이지 최상단으로 스크롤된다.
2. **Given** 헤더가 표시된 상태에서, **When** "판매 차량" 버튼을 클릭하면, **Then** 판매 차량 섹션으로 스크롤된다.
3. **Given** 헤더가 표시된 상태에서, **When** "출고 내역" 버튼을 클릭하면, **Then** 출고 내역 섹션으로 스크롤된다.
4. **Given** 헤더가 표시된 상태에서, **When** "전화번호" 버튼을 클릭하면, **Then** 전화 앱이 실행되거나 전화 걸기 다이얼로그가 표시된다.

---

### Edge Cases

- 판매 차량이 0개일 때 어떻게 표시하는가? → "등록된 차량이 없습니다" 메시지 표시
- 선택한 필터 조합에 해당하는 차량이 없을 때? → "조건에 맞는 차량이 없습니다" 메시지 표시
- 출고 내역이 0개일 때? → 섹션 자체를 숨기거나 "출고 내역이 없습니다" 메시지 표시
- 이미지 로드 실패 시? → 기본 플레이스홀더 이미지 표시
- 네트워크 오류로 데이터 로드 실패 시? → 오류 메시지와 새로고침 안내 표시 (현재는 mock data 사용으로 해당 없음)

## Requirements *(mandatory)*

### Functional Requirements

**헤더**
- **FR-001**: 시스템은 모든 화면에서 상단에 고정된(sticky) 헤더를 표시해야 한다.
- **FR-002**: 헤더 좌측에 로고가 표시되어야 하며, 클릭 시 페이지 최상단으로 스크롤되어야 한다.
- **FR-003**: 헤더 우측에 "판매 차량", "출고 내역", "빠른 상담", "전화번호" 버튼이 표시되어야 한다.
- **FR-004**: "빠른 상담" 버튼 클릭 시 견적 상담 모달이 열려야 한다.
- **FR-005**: "전화번호" 버튼 클릭 시 전화 연결(tel: 링크)이 실행되어야 한다.

**히어로 섹션**
- **FR-006**: 히어로 섹션은 배경 이미지, 타이틀, 설명, CTA 버튼으로 구성되어야 한다.
- **FR-007**: 히어로 CTA 버튼 클릭 시 견적 상담 모달이 열려야 한다.

**판매 차량 섹션**
- **FR-007.1**: 판매 차량 섹션 상단에 검색 입력창 UI를 표시해야 한다. 단, 검색 기능은 "준비 중"으로 비활성화하고 placeholder 텍스트로 안내한다.
- **FR-008**: 시스템은 1차 필터(전체/국산차/수입차)를 제공해야 한다.
- **FR-009**: 국산/수입 구분은 제조사(Manufacturer)의 category 속성으로 판단해야 한다.
- **FR-010**: 시스템은 2차 필터로 제조사 로고 카드를 가로 스크롤 또는 좌우 버튼으로 제공해야 한다.
- **FR-011**: 제조사 필터는 isVisible이 true인 제조사만 표시해야 한다.
- **FR-012**: 선택된 제조사는 시각적으로 강조(테두리/배경)되어야 한다.
- **FR-013**: 차량 카드는 썸네일, 제조사 로고/이름, 차량명, 가격, 뱃지, CTA 버튼을 포함해야 한다.
- **FR-014**: 가격이 있으면 "렌트 X원" 형식으로, 없으면 "렌트 비용문의"로 표시해야 한다.
- **FR-015**: 뱃지는 "즉시출고", "프로모션" 등 정해진 값만 표시해야 한다.
- **FR-016**: isVisible이 true인 차량만 표시해야 한다.
- **FR-017**: 차량은 sortOrder 오름차순, createdAt 내림차순으로 정렬되어야 한다.
- **FR-017.1**: 차량 카드 그리드는 반응형으로 구현해야 한다: 모바일(~767px) 1열, 태블릿(768px~1023px) 2열, 데스크톱(1024px~) 4열.
- **FR-018**: 차량 카드의 CTA 버튼 클릭 시 해당 차량 정보가 자동 세팅된 견적 모달이 열려야 한다.

**핵심 강점 섹션**
- **FR-019**: 핵심 강점 섹션은 "왜 S&C 신차장기렌트리스일까?" 타이틀과 강점 카드들로 구성되어야 한다.
- **FR-020**: 각 강점 카드는 이미지, 제목, 설명을 포함해야 한다.

**출고 내역 섹션**
- **FR-021**: 출고 내역은 2x3 그리드로 최대 6개까지 표시해야 한다.
- **FR-022**: 출고 차량 카드는 이미지, 차량명, 출고일을 포함해야 한다.
- **FR-023**: isVisible이 true인 출고 차량만 표시해야 한다.
- **FR-024**: 출고 차량은 sortOrder 오름차순, releasedAt 내림차순, createdAt 내림차순으로 정렬되어야 한다.

**FAQ 섹션**
- **FR-025**: FAQ는 아코디언 UI로 구현되어야 한다.
- **FR-026**: 한 번에 하나의 FAQ만 펼쳐져야 한다(다른 것을 펼치면 기존 것은 닫힘).

**푸터**
- **FR-027**: 푸터는 사업자 정보를 포함해야 한다.
- **FR-028**: 푸터는 개인정보처리방침 및 이용약관 링크를 포함해야 한다(placeholder 가능).

**견적 상담 모달**
- **FR-029**: 모달은 이름(필수), 전화번호(필수), 지역(필수), 유형(필수), 초기자금(필수), 계약기간(필수), 개인정보 동의(필수) 필드를 포함해야 한다.
- **FR-030**: 유형은 개인/개인사업자/법인 중 선택해야 한다.
- **FR-031**: 초기자금은 보증금(0/10/20/30%)과 선수금(0/10/20/30%) 중 택일 후 비율을 선택해야 한다.
- **FR-032**: 계약기간은 36/48/60개월 중 선택해야 한다.
- **FR-033**: 전화번호 입력 시 숫자와 하이픈을 허용하되, 제출 시 숫자만 추출해야 한다.
- **FR-034**: 모든 필수 필드와 개인정보 동의 체크 시에만 제출이 가능해야 한다.
- **FR-035**: 제출 중에는 버튼이 비활성화되어 중복 제출을 방지해야 한다.
- **FR-036**: 제출 성공 시 "접수 완료" 메시지를 표시해야 한다.
- **FR-037**: 차량 카드에서 모달을 열 때 해당 차량 정보(차량명, 제조사명)가 자동으로 세팅되어야 한다.

**데이터 구조**
- **FR-038**: 시스템은 Manufacturer, SaleCar, ReleasedCar 데이터 구조를 지원해야 한다.
- **FR-039**: SaleCar의 뱃지는 SaleCarBadge enum(즉시출고, 프로모션)만 허용해야 한다.
- **FR-040**: 데이터 소스는 교체 가능한 구조(추후 API/DB 연동 대비)로 설계되어야 한다.

**디자인**
- **FR-041**: 메인 컬러는 메탈릭 그레이(Tailwind slate/gray 계열)를 사용해야 한다(배경, 라인, 카드 테두리, 헤더).
- **FR-042**: CTA 버튼과 활성 필터는 청남색(Tailwind navy 계열, #1e3a5f 기준)을 사용해야 한다.
- **FR-043**: UI 톤은 차분하고 고급스러운 렌트/금융 느낌을 유지해야 한다.
- **FR-044**: globals.css에 프로젝트 색상 변수를 정의해야 한다: primary(청남색), secondary(메탈릭 그레이), 및 UI 상태별 색상(hover, active, disabled 등).

### Key Entities

- **Manufacturer (제조사)**: 차량 제조사 정보. 국산/수입 구분(category), 로고, 이름, 정렬순서, 노출여부 포함.
- **SaleCar (판매 차량)**: 판매 중인 차량 정보. 제조사 코드, 차량명, 썸네일, 렌트/리스 가격, 뱃지, 정렬순서, 노출여부 포함.
- **ReleasedCar (출고 차량)**: 출고 완료된 차량 정보. 차량명, 썸네일, 출고일, 정렬순서, 노출여부 포함.
- **SaleCarBadge (뱃지)**: 판매 차량에 부착되는 라벨. "즉시출고", "프로모션" 값만 허용.

- 초기 구현은 DB 없이 mock data만 사용한다.
